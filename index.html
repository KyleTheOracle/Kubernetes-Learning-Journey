Module 1: Foundations of Kubernetes

Lesson 1: What are Containers?

Containers are lightweight, standalone, executable software packages that include everything needed to run a piece of software: code, runtime, system tools, libraries, and settings.

Container vs VM

Containers share the host OS kernel while VMs include the entire OS

Containers Explained

A visual explanation of container technology

Lesson 2: Introduction to Docker

Docker is the most popular containerization platform that allows you to build, ship, and run containers.

Docker in 7 Minutes

Quick introduction to Docker concepts

Key Docker Commands

  • docker build - Build an image
  • docker run - Run a container
  • docker ps - List containers
  • docker images - List images

Lesson 3: Why Kubernetes?

Kubernetes solves the problem of orchestrating containers at scale, providing automated deployment, scaling, and management.

Benefits of Kubernetes

  • Automated scaling
  • Self-healing capabilities
  • Service discovery and load balancing
  • Automated rollouts and rollbacks
  • Configuration management

Why Kubernetes?

Understanding the need for container orchestration

Module 2: Kubernetes Architecture

Lesson 1: Cluster Components

A Kubernetes cluster consists of a control plane (master) and worker nodes that run your applications.

Control Plane Components

  • API Server - The front-end of Kubernetes
  • etcd - Distributed key-value store
  • Scheduler - Assigns workloads to nodes
  • Controller Manager - Regulates the state

Node Components

  • Kubelet - Node agent
  • Container Runtime - Docker, containerd
  • Kube-proxy - Network proxy

Lesson 2: Kubernetes Objects

Kubernetes objects are persistent entities that represent the state of your cluster.

Kubernetes Objects Explained

Understanding the basic building blocks

Key Kubernetes Objects

  • Pods - Smallest deployable units
  • Deployments - Manage pod replicas
  • Services - Network abstraction
  • ConfigMaps & Secrets - Configuration

Lesson 2: Deploying Your First Application

Learn how to deploy a simple application to your Kubernetes cluster.

Deployment Process

  1. Create a deployment YAML file
  2. Apply the configuration
  3. Expose the application
  4. Access your application

Sample Deployment YAML


apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

Module 4: Workloads & Networking

Lesson 1: Understanding Pods

Pods are the smallest deployable units in Kubernetes that can be created and managed.

Pods Deep Dive

Understanding the fundamental unit of Kubernetes

Pod Characteristics

  • Shared network namespace
  • Shared storage volumes
  • Co-located containers
  • Ephemeral by nature

Lesson 2: Kubernetes Networking

Kubernetes has a unique networking model that enables communication between pods, services, and the outside world.

Networking Principles

  • Every Pod gets its own IP address
  • Pods can communicate without NAT
  • Agents on a node can communicate with all pods
  • Services enable stable networking
